# Phase 1 — UI Design Contract

> Visual and interaction contract for the login screen, camera selection list, and auto-connect flow. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Flutter native) |
| Preset | not applicable |
| Component library | Material 3 (built-in Flutter) |
| Icon library | Material Icons (built-in Flutter) |
| Font | System default (Roboto on Android, San Francisco on macOS) |

**Rationale:** CLAUDE.md specifies "Flutter Material 3 -- default Flutter UI. Simple app with sliders, lists, and status indicators. No need for a custom design system." Phase 1 has only 2 screens (login form, camera list). Material 3 provides all needed components out of the box.

**Theme mode:** Dark only. This is a baby monitor used at night. A dark theme prevents a bright screen from waking a sleeping parent. Use `ThemeData.dark()` with Material 3 `useMaterial3: true` and a custom `ColorScheme.fromSeed()`.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, gap between form field and its error text |
| md | 16px | Default element spacing, padding inside cards, gap between form fields |
| lg | 24px | Section padding, screen horizontal padding |
| xl | 32px | Vertical gap between major screen sections (header to form, form to button) |
| 2xl | 48px | Top padding on screens |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: Touch targets minimum 48px height for all interactive elements (Material 3 default). This matters for the Connect button and camera list checkboxes -- sleepy parents at 2 AM must not miss taps.

**Implementation:** Define as constants in `lib/core/theme/spacing.dart`:
```dart
abstract final class Spacing {
  static const double xs = 4;
  static const double sm = 8;
  static const double md = 16;
  static const double lg = 24;
  static const double xl = 32;
  static const double xxl = 48;
}
```

## Typography

| Role | Size | Weight | Line Height | Usage in Phase 1 |
|------|------|--------|-------------|-------------------|
| Body | 16px | 400 (regular) | 1.5 | Form field input text, camera names, status text, error messages |
| Label | 14px | 500 (medium) | 1.4 | Form field labels, helper text, camera online/offline badge |
| Heading | 24px | 600 (semibold) | 1.2 | Screen titles: "Connect to Protect", "Select Cameras" |
| Display | 20px | 600 (semibold) | 1.3 | Not used in Phase 1 (reserved for Phase 2 monitoring view) |

**Implementation:** Use Material 3 `TextTheme` with these overrides in the dark theme:
- `bodyLarge` = Body (16/400/1.5)
- `labelLarge` = Label (14/500/1.4)
- `headlineMedium` = Heading (24/600/1.2)
- `titleLarge` = Display (20/600/1.3)

## Color

Using Material 3 `ColorScheme.fromSeed()` with a deep blue seed for a calm nighttime feel.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `surface` from dark scheme (~#121212) | Screen backgrounds, app scaffold |
| Secondary (30%) | `surfaceContainerHigh` from dark scheme (~#2B2B2B) | Card backgrounds (camera list items), form field fill, elevated surfaces |
| Accent (10%) | `primary` from seed `Color(0xFF5C6BC0)` (indigo 400) | Connect button, Start Monitoring button, selected checkbox fill, active toggle states |
| Destructive | `error` from dark scheme (~#F2B8B5 on dark) | Login error text, connection error indicators |

**Seed color:** `Color(0xFF5C6BC0)` (Indigo 400). Calming blue-purple, not stimulating. Good contrast on dark backgrounds.

**Accent reserved for (exhaustive list):**
- "Connect" button on login screen (D-01)
- "Start Monitoring" button on camera list screen (D-06)
- Selected camera checkboxes (D-04)
- Connection success indicator dot

**Status colors (derived from Material 3 scheme):**
- Online camera: `Color(0xFF81C784)` (green 300) -- small dot indicator only
- Offline camera: `Color(0xFFE57373)` (red 300) -- small dot indicator only
- Connecting/loading: accent color with animated opacity

## Interaction Contracts

### Login Form Validation
- **When:** On submit (not on every keystroke). Field-level validation when user taps Connect.
- **IP field:** Required. Show "Console IP address is required" if empty.
- **Username:** Required. Show "Username is required" if empty.
- **Password:** Required. Show "Password is required" if empty.
- **Server errors:** Map API errors to the most relevant field (D-03). Connection errors go under IP field. Auth errors go under password field.

### Camera Selection
- **Checkbox behavior:** Toggle on tap anywhere in the row (not just the checkbox).
- **Selection limit:** Allow checking up to 2 cameras. When 2 are checked, remaining unchecked rows show muted text. Tapping a third camera does nothing (no error, no toast -- the visual muting communicates the limit).
- **Previously selected (D-08):** On auto-connect, pre-check the cameras that were selected in the last session. If a previously selected camera is now offline, still show it checked but with offline indicator.

### Navigation Flow
```
App Launch
  |
  +--> [Has saved credentials?]
  |      |
  |      Yes --> Auto-connect spinner --> Success --> Camera List (pre-selected)
  |      |                                  |
  |      |                                  Fail --> Login Screen (with error)
  |      |
  |      No --> Login Screen
  |
Login Screen --> Connect --> Success --> Camera List
  |
Camera List --> Start Monitoring --> Monitoring Screen (Phase 2 placeholder)
```
