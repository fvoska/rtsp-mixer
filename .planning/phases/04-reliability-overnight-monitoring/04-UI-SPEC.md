# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4 (Reliability + Overnight Monitoring). Generated by gsd-ui-researcher on 2026-04-23. Verified by gsd-ui-checker.

**Scope recap — only three UI surfaces change in this phase:**
1. `CameraAudioCard` — render the new `reconnecting` state (D-09, D-10, D-11).
2. `MonitoringScreen` — add one new app-bar icon that opens the health summary (D-16).
3. `HealthSummaryScreen` — new screen rendering session uptime, per-camera counters, and the event log (D-13, D-15, D-17).
4. Plus the 5-minute disconnection alert notification copy (D-04). (Not in-app UI, but copy is in scope.)

Everything else in the app (login screen, camera list, settings, monitoring layout, sliders) is untouched.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (stock Flutter Material 3) |
| Preset | not applicable |
| Component library | Flutter Material 3 built-ins (`ThemeData.dark(useMaterial3: true)`) |
| Icon library | Material Icons (`Icons.*`, bundled via `uses-material-design: true` in `pubspec.yaml`) |
| Font | Roboto (Material 3 default) — no custom font declared |

**Theme source of truth:** `lib/core/theme/app_theme.dart` — `AppTheme.dark` built via `ColorScheme.fromSeed(seedColor: Color(0xFF5C6BC0), brightness: Brightness.dark)`. All color references below MUST go through `Theme.of(context).colorScheme` or the two hardcoded status constants `AppTheme.statusOnline` / `AppTheme.statusOffline`. Do NOT introduce new top-level `Color(0x...)` constants in Phase 4.

**Spacing source of truth:** `lib/core/theme/spacing.dart` — `Spacing.xs | sm | md | lg | xl | xxl` = 4 / 8 / 16 / 24 / 32 / 48. All padding/gaps in new Phase 4 widgets MUST use these tokens. No raw `const EdgeInsets.all(10)` etc.

**Rationale for "no new design system":** The app has shipped three phases on stock Material 3 dark. Introducing shadcn or a custom token system now would be scope creep that doesn't serve the core value (reliable overnight audio). Phase 4 extends the existing visual language.

## Spacing Scale

Declared values (all multiples of 4, matching `lib/core/theme/spacing.dart`):

| Token | Value | Dart constant | Usage in Phase 4 |
|-------|-------|---------------|------------------|
| xs | 4px | `Spacing.xs` | Tight gaps (spinner to text, icon to label inside event row) |
| sm | 8px | `Spacing.sm` | Compact element spacing (status dot to camera name, event row vertical padding) |
| md | 16px | `Spacing.md` | Default element spacing (card internal padding, summary header row gap) |
| lg | 24px | `Spacing.lg` | Section padding (screen-level horizontal padding, between sticky header and event list) |
| xl | 32px | `Spacing.xl` | Layout gaps (empty-state icon-to-text gap) |
| xxl | 48px | `Spacing.xxl` | Major section breaks (not used in Phase 4) |

**Exceptions:** none. Every new widget uses tokens from `Spacing`. If a value like 12px feels necessary, the executor must round to `Spacing.sm` (8) or `Spacing.md` (16), not introduce a new constant.

## Typography

Phase 4 uses exactly **4 Material 3 text styles** from `Theme.of(context).textTheme`. Do NOT declare new `TextStyle` objects; always resolve through the theme so the user's system font-size preference is respected.

| Role | Material 3 style | Effective size/weight (stock M3) | Line height | Where used in Phase 4 |
|------|-----------------|----------------------------------|-------------|----------------------|
| Body | `textTheme.bodyMedium` | 14px / w400 | 1.43 | Status label on card ("Reconnecting…"), event detail text, summary metric values |
| Label | `textTheme.bodySmall` | 12px / w400 | 1.33 | Event timestamp, event camera name suffix, summary metric labels |
| Heading | `textTheme.titleMedium` | 16px / w500 | 1.50 | Camera name on card (existing), summary section headers ("Cameras", "Events") |
| Display | `textTheme.titleLarge` | 22px / w400 | 1.27 | Session uptime value in summary header (the one prominent number) |

**Weights:** 2 only — w400 (regular, default for `bodyMedium`/`bodySmall`/`titleLarge`) and w500 (medium, used by `titleMedium`). No custom bold weights introduced.

**Rule:** Never override `fontSize` on a Material 3 style except in the existing debug `_StreamInfoPanel` pattern (which uses `fontSize: 11` for dense monospace — this is Phase 2 legacy and stays untouched). For Phase 4 content, pick the correct role; do not downsize.

**Monospace:** The event list on the summary screen uses the proportional `bodySmall` style for readability — it is NOT a log viewer. Monospace is reserved for the existing `LogScreen` (`fontFamily: 'monospace'`) which is a separate surface.

## Color

Phase 4 adheres to the 60/30/10 split inherited from the Material 3 dark scheme generated from `seedColor: #5C6BC0`.

| Role | Material 3 token | Computed (dark scheme) | Usage in Phase 4 |
|------|------------------|-----------------------|------------------|
| Dominant (60%) | `colorScheme.surface` | ~#121318 | Scaffold background of `HealthSummaryScreen`, monitoring screen background (unchanged) |
| Secondary (30%) | `colorScheme.surfaceContainerHighest` / `Card.filled` default (`colorScheme.surfaceContainerHigh`) | ~#1E1F25 | Camera card background (unchanged), summary header card, event row container |
| Accent (10%) | `colorScheme.tertiaryContainer` + `onTertiaryContainer` | amber-leaning container color from the seed | **Reserved exclusively for the `reconnecting` state** — see reserved-for list below |
| Destructive | `colorScheme.error` / `AppTheme.statusOffline` (`#E57373`) | red | Terminal `error` state on card, `streamError` / `zombieDetected` event icons |
| Positive | `AppTheme.statusOnline` (`#81C784`) | green | `playing` status dot (existing), `streamStarted` / `reconnectSuccess` event icons |
| Info | `colorScheme.primary` | indigo from seed | `connecting` `LinearProgressIndicator` (existing, unchanged), `wifiReconnected` event icon |
| Warning | `colorScheme.tertiary` | amber from seed | `reconnecting` spinner tint, `reconnectAttempt` / `wifiDropped` / `alertFired` event icons |

**Accent reserved for (explicit list — nowhere else):**
1. The amber tint on `CameraAudioCard` header when `connectionStatus == CameraConnectionStatus.reconnecting`.
2. The small `CircularProgressIndicator` spinner that appears next to the status text in the reconnecting state.
3. The event-row leading icon for warning-severity events on the summary screen (`reconnectAttempt`, `wifiDropped`, `alertFired`).

**Explicitly NOT accent (common mistakes to avoid):**
- The monitoring screen app-bar icon that opens the summary is NOT accent. It uses the default `IconButton` tint (`colorScheme.onSurface`), same as the existing video-toggle icon next to it. An accent-tinted app-bar icon would break the "accent = reconnecting only" contract.
- Summary screen section headers are NOT accent. They use `titleMedium` at the default `colorScheme.onSurface`.
- The "Monitoring just started" empty state is NOT accent. It uses `colorScheme.onSurfaceVariant` for de-emphasis.

**Amber implementation note for executors:** Material 3's tertiary role from the indigo seed (`#5C6BC0`) produces a muted amber/olive. For the reconnecting state, use `colorScheme.tertiary` for the spinner `valueColor` and `colorScheme.tertiaryContainer` at `withValues(alpha: 0.3)` for the header tint. Do NOT use `Colors.amber` / `Colors.orange` constants — stay on the theme.

## Copywriting Contract

Phase 4 has **no new primary CTA** (monitoring is started in Phase 2, foreground service in Phase 3; no destructive action exists on any Phase 4 surface — stopping monitoring is via the existing `StopMonitoringButton`, outside this phase's UI scope).

### Camera card status labels (D-11: status only, no counter, no countdown)

|--------|----------------------------------|------------|-------|
| `idle` | (no label — dot only) | — | — |

### Event log row copy contract

Each row renders in this exact layout: `[icon] {HH:mm:ss} · {cameraName or 'Session'} · {eventLabel}` with optional `{detail}` on a second line when present.

| `HealthEventType` | Leading icon | Severity (color role) | Event label copy | Typical detail copy |
|-------------------|--------------|-----------------------|------------------|---------------------|
| `monitoringStarted` | `Icons.play_circle_outline` | positive (`statusOnline`) | `Monitoring started` | `{N} cameras` |
| `monitoringStopped` | `Icons.stop_circle_outlined` | neutral (`onSurfaceVariant`) | `Monitoring stopped` | (none) |
| `streamStarted` | `Icons.play_arrow` | positive (`statusOnline`) | `Stream started` | (none) |
| `streamError` | `Icons.error_outline` | destructive (`statusOffline`) | `Stream error` | `{error message, truncated to 80 chars with ellipsis}` |
| `reconnectAttempt` | `Icons.refresh` | warning (`colorScheme.tertiary`) | `Reconnect attempt` | `attempt {N}` |
| `reconnectSuccess` | `Icons.check_circle_outline` | positive (`statusOnline`) | `Reconnected` | (none) |
| `zombieDetected` | `Icons.warning_amber_outlined` | destructive (`statusOffline`) | `Zombie stream detected` | `{reason or signal summary, e.g. "PTS stall + buffering"}` |
| `wifiDropped` | `Icons.wifi_off` | warning (`colorScheme.tertiary`) | `WiFi lost` | (none) |
| `wifiReconnected` | `Icons.wifi` | info (`colorScheme.primary`) | `WiFi reconnected` | (none) |
| `alertFired` | `Icons.notifications_active_outlined` | warning (`colorScheme.tertiary`) | `5-minute alert sent` | (none) |

**Row format rules:**
- Timestamp format `HH:mm:ss` (24-hour, no date — session is same-day in practice; if session crosses midnight, that's acceptable for v1).
- `Session` is the literal fallback label when `HealthEvent.cameraName == null` (session-wide events).
- Middle-dot separator is U+00B7 (`·`), same character as existing `_StreamInfoPanel` in `camera_audio_card.dart` for visual consistency.
- Detail line (when present) is `bodySmall` at `onSurfaceVariant`, indented by `Spacing.lg` (24px) to align under the main row's text.

### 5-minute disconnection alert (D-04)

Local notification fired via `flutter_local_notifications` — copy locked here, channel config is in RESEARCH.md Section 4.

| Element | Exact copy |
|---------|-----------|
| Notification channel name (Android Settings display) | `Camera Offline Alerts` |
| Notification channel description | `Fires when a camera has been offline for 5 minutes` |
| Notification title | `Camera offline: {cameraName}` |
| Notification body | `No audio for 5 minutes. Tap to check.` |
| Notification ticker (screen-reader / fallback) | `Camera offline` |
| Tap action | Opens app to `MonitoringScreen` (existing route; no new intent handling required) |

**Interpolation rule:** `{cameraName}` = `CameraAudioState.cameraName` (the display name already used throughout the app). Never substitute the raw camera ID or the stream URL.

**One-shot per outage cycle (D-04):** The notification is fired exactly once per transition into a ≥5-minute non-playing window. Copy does NOT escalate ("Still offline after 10 minutes" is explicitly deferred per CONTEXT.md Deferred Ideas). If the user dismisses the notification and the camera is still offline, we do NOT re-fire. When the camera returns to `playing`, the alert-fired flag resets; a subsequent outage can fire again.

### Error state copy (card `error` status — unchanged from Phase 2, documented here for completeness)

The card renders `cs.errorMessage ?? 'Stream failed'` in `statusOffline` red. Phase 4 does NOT change this copy. The reconnect supervisor still runs a retry in the background (D-02: retry forever) regardless of the `error` display — the card may show `error` briefly during the transition window before the next retry attempt flips it back to `reconnecting`. This is acceptable and intentional: `error` = "this attempt failed"; `reconnecting` = "the next attempt is running".

### Destructive actions in this phase

**None.** Phase 4 adds no destructive UI. The existing `StopMonitoringButton` (bottom of monitoring screen) is out of scope. No "Clear events" button on the summary screen in v1 — the session resets on next `startMonitoring` (D-13).

## Accessibility Contract

| Concern | Requirement |
|---------|-------------|
| Touch targets | All `IconButton`s retain default 48×48 hit area (Material 3 default). Do not use `padding: EdgeInsets.zero` on new icon buttons. |
| Semantic labels | New app-bar summary icon: `tooltip: 'Open health summary'` (serves dual purpose — visual tooltip + TalkBack label). Event row leading icons: provide a `Semantics(label: labelForType)` wrapper so screen readers announce the event type, since the icon is a visual cue. |
| Contrast | Reconnecting tertiary color on the header tint MUST maintain ≥ 4.5:1 contrast against the tint background. Spot-check: `colorScheme.tertiary` text over `tertiaryContainer @ 30%` opacity on the card's `surfaceContainerHigh` background — if this fails contrast on a specific device, executor may drop the tint to `@ 0.2` opacity but NOT change the text color. |
| Color as sole signal | Status is conveyed by dot color + text label + icon (for events). A colorblind parent can still read "Reconnecting…" — we never rely on color alone. |
| Font scaling | All text uses `Theme.of(context).textTheme.*` — scales with system font size. Do not hardcode `fontSize`. |
| Reduce motion | The reconnecting spinner is the only new animation. It respects the OS reduce-motion setting by default via Material 3's `CircularProgressIndicator`. No additional handling needed. |
