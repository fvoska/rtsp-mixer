## Component Inventory

### MonitoringScreen (replace existing placeholder)

Full-screen scaffold with two `CameraAudioCard` widgets stacked vertically.

**Primary visual anchor:** The two `CameraAudioCard` widgets are the focal point of this screen. All other elements (AppBar, stop button) are secondary navigation chrome.

**Layout:**
```
AppBar: "Monitoring" + back arrow
|
| [CameraAudioCard #1]     <- Spacing.lg gap
| [CameraAudioCard #2]     <- Spacing.lg gap
|
| [Stop Monitoring button]  <- bottom safe area
```

**States:**
1. **Loading** -- Both cards show `CircularProgressIndicator` with "Connecting..." text while RTSP streams initialize
2. **Playing** -- Both cards show live status, volume slider, pan slider
3. **Partial error** -- One card shows error, other continues playing
4. **Full error** -- Both cards in error state

### CameraAudioCard

A `Card` widget (Material 3 filled card) containing per-camera controls.

**Layout (vertical stack within card):**
```
[Status dot] [Camera Name]           [Mute icon button]
[Volume icon] [------- Volume Slider -------] [100%]
[L]           [------- Pan Slider ----------] [R]
```

**Specifications:**
- Card uses `surfaceContainer` background via Material 3 `Card.filled()`
- Card padding: `Spacing.md` (16px) all sides
- Card corner radius: 12px (Material 3 default for filled cards)
- Status dot: 8x8px circle, same as camera list screen pattern
- Camera name: `titleMedium` weight
- Volume slider: `Slider` widget, min 0.0, max 100.0, divisions: 100
- Volume readout: right-aligned `bodySmall`, format as integer percentage "75%"
- Pan slider: `Slider` widget, min -1.0, max 1.0, divisions: 20 (0.1 increments)
- Pan labels: "L" (left) and "R" (right) in `bodySmall`, flanking the slider
- Mute button: `IconButton` with `Icons.volume_up` (unmuted) / `Icons.volume_off` (muted)
- Mute toggles volume between 0 and the previously set value

**Interaction contracts:**
- Volume slider: `onChanged` calls `Player.setVolume()` -- continuous (no debounce, media_kit handles it)
- Pan slider: `onChanged` calls `applyPan()` via NativePlayer -- continuous
- Pan slider center snap: no snap behavior (user controls precisely)
- Mute button: single tap toggles, stores previous volume for restore

### StopMonitoringButton

Bottom-positioned `OutlinedButton` (not filled -- lower visual weight than "Start Monitoring").

**Specifications:**
- Full width, 48px height (matches "Start Monitoring" pattern from camera list)
- Label: "Stop Monitoring"
- Wrapped in `SafeArea` + `Padding(horizontal: Spacing.lg, vertical: Spacing.md)`
- On press: disposes both Players, navigates back to camera list

| Element | Copy |
|---------|------|

| Pan label right | "R" |

**Destructive actions:**
- "Stop Monitoring" -- no confirmation dialog. Stopping audio is low-cost and immediately reversible (user can restart from camera list). Matches the "2-tap interaction" principle from requirements.

## State Machine: Per-Camera Audio

```
[idle] --startMonitoring--> [connecting]
[connecting] --streamOpened--> [playing]
[connecting] --error--> [error]
[playing] --streamDropped--> [error]
[error] --retry (Phase 4)--> [connecting]
[playing] --stopMonitoring--> [idle]
[error] --stopMonitoring--> [idle]
```

Visual mapping:
- **idle**: not shown (screen not visible)
- **connecting**: spinner + "Connecting..." text, sliders disabled (greyed out)
- **playing**: green dot + "Live" text, sliders enabled
- **error**: red dot + error message text, sliders disabled
