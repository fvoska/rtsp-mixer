# Quick Task 260724-83t: Add About Page with Licenses, Version Info Summary

Added an About page (version via package_info_plus, bundled CHANGELOG.md, open-source licenses via showLicensePage, and Filip Voska attribution + contact email), reachable from Settings as a stacked detail page with its own AppBar and back button.

## What Was Built

- **pubspec.yaml**: Added explicit `package_info_plus: ^10.1.0` dependency (was transitive-only at resolved 10.1.0) and declared `CHANGELOG.md` under `flutter/assets` so it loads at runtime via `rootBundle`.
- **AboutScreen** (`lib/features/about/screens/about_screen.dart`): A `StatelessWidget` mirroring the HelpScreen layout (Scaffold + AppBar('About') + centered `ConstrainedBox(maxWidth: 640)` scroll view using `Spacing.*`). Four sections:
  1. **Version** — `FutureBuilder<PackageInfo?>`; shows "Version {version} (build {buildNumber})", "Version …" while loading, or "Version unavailable" on failure.
  2. **Made by** — "Made by Filip Voska" + `SelectableText('filip.voska@infinum.com')`.
  3. **Changelog** — `FutureBuilder<String?>` over `rootBundle.loadString('CHANGELOG.md')`; raw text in a `SelectableText`, or "Changelog is not available." fallback.
  4. **Legal** — a `ListTile` invoking `showLicensePage(applicationName: 'RTSP Mixer', applicationVersion: <version-if-available>)`.
- **Routing** (`app_router.dart`): Registered top-level `GoRoute(path: '/about', builder: (_, _) => const AboutScreen())` above the `StatefulShellRoute`, next to `/help`.
- **Settings entry point** (`settings_screen.dart`): Added an `Icons.info_outline` "About" `ListTile` (with dividers) between "Help & setup guides" and "Sign out / Reset setup", pushing `/about`.

## Defensive Behavior (CLAUDE.md)

Both async reads are wrapped so failures resolve to a fallback state instead of throwing: `_loadPackageInfo()` catches and returns null; `_loadChangelog()` catches and returns null. The page always renders regardless of package_info or asset-load failure.

## Deviations from Plan

None affecting scope. One minor design choice: About sections use a lightweight titled `Card` (icon header + content) instead of the HelpScreen `ExpansionTile` accordion, because About content is short and better shown always-expanded. Layout scaffolding (Scaffold/AppBar/ConstrainedBox/Spacing) matches HelpScreen exactly as specified.
